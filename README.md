#  🍫 О проекте 
Проект реализует микросервисную архитектуру. Всего 9 полноценных сервисов (user-service, support-service, review-service, order-service, media-service, email-service, catalogue-service и т.д.)
Каждый сервис является независимым Spring Boot приложением
со своей domain-моделью и конфигурацией.

Взаимодействие между сервисами осуществляется исключительно
по сети (HTTP / gRPC/ Kafka). Сервисы используют библиотеку, из которой делят общие бины, Dto, Exception и т.д.

Для упрощения инфраструктуры используется общий экземпляр
PostgreSQL и Redis, при этом каждый сервис логически изолирует
свои данные и может быть без изменений перенесён
на собственную БД.


# ✔️ Установка и запуск проекта

### Требования 
(что должно быть установлено на Вашем ПК)
- Git
- Docker + Docker Compose

---

### Установка 
(на примере Windows 10 / 11)

1. Клонируйте репозиторий:

```
git clone https://github.com/allMyDays/my_shop.git
```
2. Откройте скачанную папку "my_shop" и переименуйте файл окружения:
```
rename .env.example .env 
```
3. Откройте файл .env и заполните переменные, не имеющие дефолтных значений
(инструкция по получению необходимых значений приведена ниже).

4. Запустите проект (Docker engine должен быть запущен):
```
docker-compose up -d
```
5. Дождитесь завершения сброки и запуска контейнеров (обычно занимает минут 10 при первом запуске).

6. После полного запуска система будет доступна по адресу:
```
http://localhost:58087/
```
(после завершения запуска контейнеров рекомендуется подождать еще минуты 2, пока все сервисы полностью прогрузятся, прежде чем переходить по адресу)

# ✔️ Получение значений для переменных, не имеющих дефолтных значений
(это переменные «MAIL_HOST», «MAIL_ADDRESS», «MAIL_PASSWORD», «GEOAPIFY_TOKEN»)

## Получение токена от сервиса Geoapifi
(Получить токен очень легко)

Для того, чтобы приложение могло предлагать вариации адреса проживания на основе строки пользователя, нормализировать этот адрес, рассчитывать расстояние и время доставки заказов, нужно получить токен от сервиса Geoapify.com и вставить этот токен как значение параметра «GEOAPIFY_TOKEN».

1) Для получения токена нужно перейти на сайт https://www.geoapify.com/ , зарегистрироваться через свой email-адрес (с подтверждением адреса).
2) Нажать "Create project", придумать любое имя проекту, скопировать созданный токен и вставить токен как значение параметра «GEOAPIFY_TOKEN» в файле ".env".

## Настройка отправки почты (SMTP)

Для того, чтобы приложение могло отправлять код подтверждения при регистрации, смене пароля или email-адреса, приложению необходимо уметь отправлять электронные письма через SMTP.

Для этого нужно указать:
- адрес SMTP сервера почтового сервиса,
- Ваш email, с которого будут отправляться письма,
- специальный пароль от вашего email-адреса для внешних приложений.

### Переменные окружения

В файле `.env` необходимо указать следующие параметры:

MAIL_HOST — адрес SMTP сервера  
MAIL_ADDRESS — Ваш email адрес, с которого будут отправляться письма  
MAIL_PASSWORD — пароль от вашей почты для внешних приложений (SMTP)

---

### Примеры SMTP-хостов

**Mail.ru**
MAIL_HOST=smtp.mail.ru

**Gmail**
MAIL_HOST=smtp.gmail.com

**Yandex**
MAIL_HOST=smtp.yandex.ru

Приложение использует защищённое соединение **SMTPS (порт 465)** автоматически.

---

⚠ Используется **НЕ обычный пароль от почты**, а **специальный пароль для внешних приложений**.

### Как получить пароль для SMTP на примере сервиса Mail.ru 
 (бесплатно, без платной подписки)

1. Войдите в аккаунт Mail.ru
2. Перейдите в **Настройки → Безопасность**
3. Создайте **пароль для внешних приложений (SMTP)**
4. Скопируйте полученный пароль и используйте его в `MAIL_PASSWORD`

Обычный пароль от почты использовать **нельзя**.

### Как получить пароль в сервисе gmail.com 
(бесплатно, без платной подписки!)

1. Здесь без включенной 2-этапной аутентификации ключ создать нельзя, поэтому нужно перейти по ссылке https://myaccount.google.com/security , найти "Двухэтапная аутентификация" -> нажать "Включить". 

2. Создать нужный нам ключ, перейдя по ссылке https://myaccount.google.com/apppasswords , ввести любое имя приложения (например, "spring-mail"), нажать "Создать". Гугл покажет 16-значный пароль с пробелами, но вставлять ключ в .env файл нужно без пробелов. 

---

### Пример `.env`

```
MAIL_HOST=smtp.mail.ru
MAIL_ADDRESS=your_email@mail.ru
MAIL_PASSWORD=your_smtp_password
```

или 
```
MAIL_HOST=smtp.gmail.com
MAIL_ADDRESS=your_email@gmail.com
MAIL_PASSWORD=your_smtp_password
```
